<h2>★“电子书清单”改版通知</h2><br />&#12288;&#12288;先插播一个通告：<br />&#12288;&#12288;这几年来，俺一直采用 Google Code 的 wiki 功能来汇总电子书清单。<br />&#12288;&#12288;最近2周，俺重新调整了<a href="https://code.google.com/p/program-think/wiki/Books" target="_blank">电子书清单</a>的排版。把表格中的“备注”那一列的内容单独放到 Google Docs 里面。表格中只放一个超链接。这样可以避免备注的内容太多，导致表格的排版很难看（把其它几列挤得很窄）。<br />&#12288;&#12288;另外一个考虑是：由于 wiki 页面的内容越来越多，会触及 wiki 的上限（Google Code 对 wiki 页面的代码总长度设有上限）。把备注挪走之后，wiki 内容的总长度大大减少了。<br />&#12288;&#12288;可能有读者会提建议说：把不同分类的电子书放到不同的 wiki 页面。但俺觉得这样不好。当初俺把所有书目放到同一个页面，是为了方便大伙儿在页面内查找（用浏览器的 Ctrl + F）。如果分成多个页面，就没法用浏览器进行页面内查找了 :(<!--program-think--><br /><br /><h2>★“分享电子书的博文”改版通知</h2><br />&#12288;&#12288;过去2-3年里，俺每次分享电子书，都是在博文中列出“新增的电子书”的书名和介绍。大伙儿如果要下载，还要重新打开 Google Code 上的“<a href="https://code.google.com/p/program-think/wiki/Books" target="_blank">电子书清单</a>”这个页面。<br />&#12288;&#12288;今后俺如果发布电子书更新通知，会在博文中列出所有的电子书（含网盘链接）。对于近期新增加的电子书，用醒目的颜色标注（目前采用：<span style="background:lightgreen;">绿色填充表格中对应的那行</span>）。<br />&#12288;&#12288;这样可以带来两个好处：<br />&#12288;&#12288;<b>好处1：</b><br />&#12288;&#12288;你如果要下载，就不用再打开“<a href="https://code.google.com/p/program-think/wiki/Books" target="_blank">电子书清单</a>”这个页面，降低大伙儿点击鼠标的次数。<br />&#12288;&#12288;<b>好处2：</b><br />&#12288;&#12288;因为俺已经提供了“博客打包下载”的功能。所以今后定期发布的“更新通知”，也会包含在打包电子书内。<br />&#12288;&#12288;如果你下载了博客打包的电子书，不用联网就可以查看俺网盘上的书目清单。<br /><br /><hr><font size="4"><b>再次罗嗦一下：<br
